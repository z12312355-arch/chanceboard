# 奇蹟之盤 Chanceboard — 強度驗證工具包

這是我用來驗證角色強度平衡的自動化測試工具，搬到你自己電腦上跑，理論上會比我這邊的雲端環境（只有2顆CPU核心）快，因為大部分電腦的核心數都比較多。

## 這個資料夾裡有什麼

- `run-balance-test.bat` — **最簡單的用法，雙擊這個檔案就好**，不用碰終端機、不用背指令。第一次執行會自動幫你裝好需要的套件，之後每次雙擊會問你要跑幾場、開幾個分頁（直接按Enter用預設值也可以），跑完會自動整理好統計資料、自動用瀏覽器打開報表。詳見下面「最簡單的用法」。
- `miracleboard_testharness.html` — 目前這個時間點的遊戲檔案副本，純粹放這裡當作跑批次的預設對象，方便你不用每次都打參數指路徑。**現在戰鬥模擬用的技術性入口已經直接寫進遊戲主程式碼裡了，不再是外加的特製版本**——這代表你自己在玩的 `chanceboard.html`，或是之後從遊戲後臺「下載完整遊戲檔案」匯出的任何新版本，都可以直接拿來跑這套批次驗證，不用再跟我要新檔案（詳見下面「自己更新角色資料」）。
- `run_balance_trials.js` — 跑批次戰鬥的主腳本（`run-balance-test.bat`背後呼叫的就是這支）。
- `analyze_balance.js` — 把跑出來的結果統計成各角色的勝率表格（含勝率、場均輸出/承傷/回血、KO/陣亡），同時會另外產生一份`balance_report.html`給你用瀏覽器看更好讀的版本。
- `balance_results.jsonl` — 範例輸出（我這邊先跑了幾場測試留著給你參考格式，你自己跑之後會被覆蓋）。

## 最簡單的用法（推薦）

直接雙擊`run-balance-test.bat`。

（原本第一版做成中文檔名`一鍵跑強度驗證.bat`，結果因為Windows終端機的編碼問題，中文內容被讀成亂碼、整個檔案跑不動——這版改成純英文內容避開這個問題，經驗證是純ASCII編碼，不會再有這個狀況。跑起來的訊息是英文，但實際跑戰鬥產生的中文資料、統計報表都完全不受影響，一樣是中文。舊的那個中文檔名`.bat`可以直接刪掉。）

第一次執行會先自動安裝需要的套件（可能要等一兩分鐘，之後就不用再等了）。接著會問你兩個問題（英文）：

```
How many games to run? (press Enter for default 360):
How many parallel browser tabs? (press Enter for default 4, match your CPU core count):
```

不知道要填什麼的話，直接按兩次Enter用預設值就好（360場、4個分頁）。接下來它會自己跑完戰鬥、自己統計結果、自己用瀏覽器打開報表給你看，全程不用再輸入任何指令。跑完視窗會停在那裡，按任意鍵或直接關掉都可以。

如果雙擊沒反應或跳出奇怪的錯誤訊息，把畫面截圖傳給我，我幫你看。

## 手動用法（進階／想自己控制細節時用）

## 重要：`chanceboard.html` 現在需要搭配 `chanceboard_db.js`

2026年7月這次更新之後，你的正式版遊戲資料夾裡（`chanceboard.html` 旁邊）多了一個 `chanceboard_db.js` 檔案——這才是**真正的資料庫**，遊戲每次開啟都會直接讀取這個檔案「當下」的內容（不再是把資料整個寫死在html裡）。這代表：

- 這兩個檔案要放在同一個資料夾裡，`chanceboard.html` 才抓得到資料。
- 只複製 `chanceboard.html` 單獨帶著走（例如傳給朋友）也不會壞掉，只是會自動改用內建在html裡的備援資料，不是最新資料——如果要連最新資料一起帶走，記得兩個檔案一起複製，或用後臺的「下載完整遊戲檔案」另外匯出一份單一檔案版本。
- 這支批次驗證腳本（`run_balance_trials.js`）預設指向 `../chanceboard.html`，只要它旁邊的 `chanceboard_db.js` 也在，跑批次時會自動一起讀到，不用額外設定。

## 自己更新角色資料，不用每次都跟我要

現在有兩種方式，看你想改整批資料還是只改一兩個數值：

**方式A：直接編輯資料庫檔案（推薦，最快）**

1. 用文字編輯器（記事本、VSCode都可以）直接打開 `chanceboard_db.js`，找到想改的角色/招式，把數值或文字改掉，存檔。
2. 打開 `chanceboard.html` 玩玩看，或執行 `node run_balance_trials.js 360 8` 直接跑驗證——因為檔案是每次開啟都重新讀取，改完存檔就直接生效，不用透過後臺介面，也不用重新下載整個html。
3. 這個檔案最外層是 `window.CHANCEBOARD_DB = { ... };`，這層包裝不要動，只改大括號裡面的內容就好；`effectOps` 那種一層一層包起來的欄位比較適合交給後台介面的JSON編輯欄位改（下面方式B），純數值/文字欄位（像HP、威力比、名稱、描述）直接在這裡改最快。

**方式B：透過遊戲內的後臺介面改，再匯出成資料庫檔案**

1. 打開 `chanceboard.html`，點右下角的後臺／管理面板按鈕（⚙️），把想調整的角色/招式數值改一改，按「套用」。
2. 按後臺的「下載資料庫檔案 (chanceboard_db.js)」，下載下來的新檔案直接蓋掉你資料夾裡原本那份 `chanceboard_db.js` 即可（不用整個html都重新下載）。
3. 執行 `node run_balance_trials.js 360 8` 就會用你剛剛改過的數值去跑驗證。

（如果想要一份完全獨立、不需要另外帶 `chanceboard_db.js` 的單一檔案，方便分享給別人，後臺還是保留原本的「下載完整遊戲檔案」按鈕可以用，效果跟以前完全一樣。）

原理是：戰鬥模擬用的入口已經是遊戲主程式碼的一部分（不是我另外注入的），所以不管是直接編輯 `chanceboard_db.js`、透過後臺套用、或匯出的任何新版本，都可以直接被這支腳本使用。**唯一要注意的是**：如果你拿的是2026年7月這次更新「之前」就已經存在電腦裡的舊版game html（沒有這個入口、也不認得外部資料庫檔案），會報錯——這種情況下，打開那份舊檔案的後臺，用「下載完整遊戲檔案」重新匯出一次最新版本即可。

## 事前準備（只需要做一次）

1. **安裝 Node.js**：如果還沒裝過，到 https://nodejs.org 下載LTS版本安裝（Windows直接下一步到底即可）。安裝好之後打開「命令提示字元」或「PowerShell」，輸入 `node -v` 確認有顯示版本號。

2. **安裝 Playwright 套件**：在這個資料夾（`balance-test-tools`）裡打開終端機，執行：
   ```
   npm install playwright
   npx playwright install chromium
   ```
   第一次執行會下載瀏覽器元件，可能要等一下。

## 怎麼跑

在這個資料夾裡執行：

```
node run_balance_trials.js 360 8
```

- 第一個數字（360）是要跑幾場，數字越大結果越準，但也越花時間。想快速看一下可以先跑 `20`；正式要做平衡判斷建議 `300` 以上。
- 第二個數字（8）是同時開幾個瀏覽器分頁平行跑。這個數字可以抓「你電腦的CPU核心數」，例如8核心的電腦可以填 `8`，數字抓太高反而可能因為搶資源變慢，可以自己抓幾個數字比較看看。

跑完之後會看到類似這樣的結尾訊息：

```
=== DONE ===  trials: 360 not-ended: 0 total errors: 0 elapsed: 245.3s per-game avg: 0.681s
結果已寫入： .../balance_results.jsonl
```

接著執行：

```
node analyze_balance.js balance_results.jsonl
```

就會印出每個角色的場次、勝率、場均輸出/承傷/回血、KO/陣亡數，由勝率高到低排序，同時會多產生一份`balance_report.html`，用瀏覽器打開會看到排版更整齊、勝率過高標紅過低標綠的網頁表格。

## 小提醒

- 每場對戰是隨機從21位角色裡各挑3人組隊，場次越多，每個角色被抽到的次數越平均，統計結果也越可靠；場次太少（例如20~30場）常常會因為隨機性造成數字大幅波動，不能太當真。
- 場均回血這欄的統計方式：有標明施術者的回血（例如某角色的被動幫隊友回血）算在施術者身上；沒標施術者的（例如自己觸發的被動回血、花牌回復）算在receiver自己身上；這是從文字紀錄反推出來的合理估計，不是100%精確的逐筆歸因，數字大方向可信但不用太計較個位數差異。
- 這份工具只是我平常用來輔助判斷平衡性的參考數據，不代表最終定案，實際要不要調整還是以你的判斷為準。
